# Portfolio Packaging

## Positioning

### Short project description

Built a collaborative dashboard builder with drag-and-drop layout editing, interactive chart widgets, global filtering, autosaved persistence, and realtime multiplayer presence with live cursors.

### Technical framing

Designed a frontend architecture that separates:

- ephemeral UI state
- persisted server state
- derived filtered datasets
- room-based realtime presence

This keeps interaction-heavy experiences responsive while maintaining backend simplicity.

## Honest Effort Estimate

- essential polished MVP: 2-3 focused weeks
- polished with strong demo assets/readme: 3-4 weeks

## README Talking Points

Explicitly explain:

- why scope is intentionally constrained
- frontend state architecture decisions
- why realtime is best-effort
- where backend complexity was intentionally avoided

## Final Scope Statement

The finished project should be:

- frontend-led
- interactive
- architecturally thoughtful
- polished enough to demo confidently
- scoped to finish

It should not become:

- a mini BI platform
- a query engine
- a full collaborative editing system
- another backend-heavy project
